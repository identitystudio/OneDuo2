## Design Principles (from feedback)

1. **Less words, more clarity** - Every word must earn its place
2. **Show, don't explain** - The interface should be self-evident
3. **One path forward** - Don't overwhelm with options
4. **Execution reality** - Focus on "what do I DO next"

---

## Priority Order

1. Dashboard simplification (where users spend most time)
2. Processing status clarity (reduces support questions)
3. Copy cleanup (systematic jargon removal)
4. Navigation simplification
